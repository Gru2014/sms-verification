# Verification and OTP Message Examples

Use these templates for UI text, help docs, and support snippets.

## OTP delivery messaging

**Example A**
`Code sent. Most messages arrive within 1-3 minutes. Keep this page open.`

**Example B**
`Waiting for SMS code. If it does not arrive, confirm your VPN/IP matches the selected number country.`

## "SMS not received" support replies

**Example A**
`Please confirm your IP location matches the number country, then request the code again. Region mismatch is the most common cause of failure.`

**Example B**
`If the code has not arrived after the full wait window, try a higher-tier number or switch to a different country for this service.`

## One-time number guidance

**Example A**
`For one-time activations, if no code arrives in time, cancel and purchase a new number to retry.`

**Example B**
`One-time numbers are best for single verifications. Use rental numbers if you expect multiple code requests.`

## Rental number guidance

**Example A**
`Your rental is active. Use "Request another SMS" to receive a new code on the same number.`

**Example B**
`Rental numbers let you keep the same line during the lease period, which helps when a platform asks for another verification code.`

## Security and trust messaging

**Example A**
`Never share your OTP code with anyone, including support.`

**Example B**
`TrustSMSS support will never ask for your full verification code.`

## Messaging best practices

- Keep support instructions short and actionable
- Prioritize region-match checks early
- Clearly distinguish one-time vs rental behavior
- Avoid ambiguous retry instructions
- Include expected wait windows in every flow
